# 📄 ZMODEM 架构收敛规范（OpenCode 任务说明）

## 🎯 目标

将当前 ZMODEM 实现从：

> “耦合式状态驱动实现”

收敛为：

> “三层职责严格分离的协议引擎”

核心目标：

* 消除 parser / core / mux 的控制流耦合
* 强制 CRC 校验闭环
* 消除 silent failure / infinite state
* 使系统具备可测试 deterministic behavior

---

# 1. 架构分层（强制约束）

## 1.1 Parser 层（Byte → Frame）

### 职责

Parser **只做一件事**：

> 将 byte stream 转换为 protocol frame

### 禁止行为 ❌

* ❌ 不得读取 core state
* ❌ 不得使用 `_expectDataSubpacket`
* ❌ 不得决定协议流程
* ❌ 不得吞没 header / frame
* ❌ 不得 silent discard CRC

### 必须行为 ✅

* decode ZMODEM frame（HEX / BIN / DATA）
* 输出统一 Frame 结构：

  ```dart
  ZFrame {
    type,
    header,
    data,
    crc,
    isValidCRC
  }
  ```

---

## 1.2 Core 层（State Machine Owner）

### 职责

> 唯一拥有 ZMODEM 状态机逻辑

### 必须拥有

* state machine transition table
* CRC validation decision
* timeout control
* error recovery policy

### 禁止行为 ❌

* ❌ 不直接 parse bytes
* ❌ 不处理 byte stream
* ❌ 不绕过 parser

---

## 1.3 Mux 层（Routing Layer）

### 职责

> 只做事件路由，不参与协议逻辑

### 允许

* session lifecycle
* terminal mode switch
* event forwarding

### 禁止 ❌

* ❌ 修改 parser 行为
* ❌ 修改 core state
* ❌ buffer manipulation affecting protocol

---

# 2. CRC 强制校验（必须修复）

## 当前问题

* CRC 被读取但未校验（C1–C3）

## 规范

### 每个 Frame 必须：

```text
parse → compute CRC → validate → reject OR accept
```

### 行为规则：

| CRC结果   | 行为               |
| ------- | ---------------- |
| valid   | 交给 core          |
| invalid | 丢弃 + error event |

❗ 禁止 silent accept

---

# 3. State Machine 规范（Core）

## 3.1 必须显式定义状态

所有 state 必须在 enum / class 中声明

禁止 implicit state（如 flag 驱动 state）

---

## 3.2 状态迁移必须集中管理

```text
handleEvent(event) → transition(state, event)
```

❌ 禁止：

* state scattered in methods
* hidden transitions inside parser

---

## 3.3 必须增加 timeout model

每个 blocking state 必须有：

```dart
timeout → recovery transition
```

最低要求：

* WAITING_ZDATA
* WAITING_ZEOF
* WAITING_ZFIN
* WAITING_REMOTE_RESPONSE

---

# 4. Parser DATA mode 重大 bug（必须修）

## 当前问题

DATA mode 会吞掉 header frame（T1 / T4）

## 原则修复

### Parser 必须：

> 永远不能基于 core state 改变解析行为

### 修复目标：

* header frame 在任何 mode 都必须被识别
* DATA subpacket 与 header frame 必须严格区分

---

# 5. Buffer / Memory 约束

## 必须修复

* BytesBuilder 无限制增长（M3）
* CSI params 无限制（P3）

## 规范

```text
所有 buffer 必须有上限
超过 → drop + error event
```

建议默认：

* frame buffer: 64KB
* CSI params: 256

---

# 6. Mux 层修复要求

## 关键问题

* async void (_handleZModem)（M1）
* backpressure disabled（M2）
* null file offer leads to stuck session（M5）

## 必须修复

### 6.1 async void 禁止

```dart
Future<void> handleZModem()
```

---

### 6.2 backpressure 必须恢复

* stream must not grow unbounded
* pause/resume required

---

### 6.3 session must always terminate

规则：

```text
if no file offer → must send ZFIN or timeout exit
```

---

# 7. Testing Requirements（强制）

## 必须新增测试类型：

### 7.1 Fuzz test

* random byte stream
* corrupted frames
* partial packets

目标：

> no crash, no infinite loop

---

### 7.2 Protocol compatibility test

必须测试：

* lrzsz
* SecureCRT / XShell（任一）

---

### 7.3 soak test

```text
1000 file transfers loop
```

# 8. 允许的修改范围（非常重要）

OpenCode 只能：

### ✔ allowed

* refactor into 3 layers
* fix CRC validation
* fix parser DATA mode bug
* add timeout handling
* fix mux lifecycle bugs
* add tests

### ❌ forbidden

* feature redesign
* protocol redesign
* changing ZMODEM spec interpretation
* UI/terminal unrelated refactor
* “clean up architecture” without bug rationale

---

# 9. 成功标准（Definition of Done）

系统必须满足：

* [ ] parser independent of core state
* [ ] CRC enforced (no silent accept)
* [ ] no infinite state possible
* [ ] all blocking states have timeout
* [ ] fuzz test runs 1M iterations without crash
* [ ] no unbounded buffer growth
* [ ] mux cannot affect protocol correctness

> Parser parses
> Core decides
> Mux routes

👉 视为架构 bug，而不是实现细节
